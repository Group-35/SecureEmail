Artifact Size Metric Standard

Our project will be broken down into many two main sections as follows: Server and Client.

For the Server section, we have these 2 portion to work on:

  • Networking: Networking shall strive to create software that will run reliably with only 1 hour maintenance period per calendar month.
    • The Size metric will not be able to be tested without a month of time to test, however the project is over before a month after completing the project.
  • Encryption: Encryption shall achieve 128-bit level of encryption that requires less than .5 seconds to encrypt and decrypt.
    • The Size metric for this will be measured by acquired certificates or acquired algorithms.

For the Client section, we have these 2 portion to work on:

  • GUI: Simple, Clean, and Effective. The GUI will strive to be easy enough for a first time user to understand within 1 minute.
    • The Size metric for this will be measured by group consensus as the testing period will be too short for any other testing.
  • Features: Features will be measured by the quantity of quality features beyond sending and receiving.
    • The Size metric for this will be measured by group consensus and requirements listed as the testing period will be too short for any other testing.
