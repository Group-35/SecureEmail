SECTION 1: Introduction

Software to be Produced:

The current system design is an encrypted email system through a web browser based interface. Messages will be encrypted end-to-end so that they can only be read by the designated receiver. Users will be able to log in and out securly. They will also be able to send andreceive messages, organize their messages, mark them as spam, and manage their address book.

Reference Documents:

  • Concept of Operations
  • Project Plan
  • Test Plan

Applicable Standards:

  • The Advanced Encryption Standard (http://csrc.nist.gov/groups/STM/cavp/documents/aes/aesval.html)
  • Ubuntu 14.04 LTS
  • Current generation web browsers

SECTION 2: Product Overview

Assumptions:

  • The User will have basic understanding of email programs
  • The User will have basic understanding of web browsers
  • The Users device is connected to internet
  • The Web Application will run on current generation web browsers
  • The Web Application will be stable

Stakeholders:

  • Professor and TAs
    • These stakeholders will be analyzing the group information and development process throughout the course. They will also be grading us based upon said documentation and through presentations of the Web Application.
  • Software Developers
    • These stakeholders will be responsible for the maintenance and handling of any problems, bugs, or fixes that will be needed before, during, or after the release of the Web Application.
  • Users
    • These stakeholders will be relying on the stability, reliability, and security of the Web Application.

Use Case Diagram:

Use Case Descriptions:

The Cases in this diagram are described as follows.

         Log in is used when the user logs in.

         Log out extends from this case, and allows the user to log out.

         Compose Email, this will allow the user to pull up a blank email.

o    Extended from this is the save draft case, which allows the user to save a pre written email that has not been sent.

o    Included in this is the Send email case, which allows the user to send the email that they wrote.

         View Email case allows the user to view the emails that have been sent to them.

o    Extended from this is the Organize Email case, which allows the user to organize their inbox by either, time, user, or if the message is read or not.

o    Another case extended from View email is Reply to message, this allows the user to reply to a message that was sent to them.

o    Included in View email is Mark as read, this means that whenever a user reads a message, it will automatically be marked as read.

         Create account case allows new users to create an account to use.

         Edit Profile case which allows the user change certain information associated with their account.

         View Contact List case allows the user to view people they have entered into their contact list.

o    Extended from this is the Edit Contacts case which allows the user to edit the contacts in their contact list.
